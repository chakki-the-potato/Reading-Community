# 책책책 - 독서 모임 관리

매일 10분 독서 인증과 월 1권 완독을 목표로 하는 독서 모임 관리 웹앱입니다. 카카오톡 채팅 내보내기를 파싱하여 인증 기록을 자동으로 생성하고, 벌금을 계산합니다.

## 주요 기능

- **매일 독서 인증** - 사진 증거 + 3줄 이상 감상문으로 인증
- **인증 기록 조회** - 분기별 인증 현황을 테이블로 확인
- **벌금 자동 계산** - 미인증 일수에 따른 벌금 자동 산출
- **납부 현황 관리** - 멤버별 벌금 납부 여부 추적
- **카카오톡 파싱** - PC/Android/iOS 내보내기 형식 모두 지원

## 기술 스택

| 구분 | 기술 |

## 시작하기

### 웹페이지

`index.html`을 웹 서버를 통해 열면 인증 기록을 조회할 수 있습니다.

### 인증 확인 스크립트

카카오톡 채팅 내보내기 파일을 파싱하여 인증 기록을 생성합니다.

```bash
python3 check_verification.py \
  --members members.csv \
  --start 2026-02-01 \
  --end 2026-02-15 \
  --output records/2026/02/verification_record.csv \
  --label "2월 1분기" \
  chat_export.txt
```

#### 옵션

| 옵션 | 설명 |
|------|------|
| `chat_files` | 카카오톡 채팅 내보내기 파일 (.txt 또는 .csv) |
| `--members, -m` | 멤버 목록 CSV 파일 (필수) |
| `--start, -s` | 시작일 YYYY-MM-DD (필수) |
| `--end, -e` | 종료일 YYYY-MM-DD (필수) |
| `--output, -o` | 출력 CSV 경로 (선택) |
| `--exclude, -x` | 인증 제외일 (선택) |
| `--label, -l` | 기록 라벨, 예: "2월 1분기" (선택) |

### 인증 조건

하루 인증이 유효하려면 같은 날(오전 5시 기준) 아래 두 가지를 모두 충족해야 합니다:

1. **사진** - "사진", "Photo", "동영상" 등의 메시지
2. **감상문** - 첫 줄에 '읽' 포함, 최소 3줄 이상

## 벌금 규칙

| 미인증 일수 | 벌금 |
|-------------|------|
| 1~5일 | 일당 2,000원 (최대 10,000원) |
| 6일 이후 | 일당 1,000원 추가 |

예시: 10일 미인증 = 10,000원 + (5 x 1,000원) = **15,000원**

## 운영 매뉴얼

### 1단계 — 분기 종료

#### 1-1. 인증 기록 생성

카카오톡 채팅 내보내기 파일을 준비한 뒤 스크립트를 실행합니다.

```bash
python3 check_verification.py \
  --members members/{YYYY}/{MM}/members_{월}{분기}.csv \
  --start YYYY-MM-DD \
  --end YYYY-MM-DD \
  --output records/{YYYY}/{MM}/verification_record_{시작일}_{종료일}.csv \
  --label "{월} {분기}" \
  chat_export.txt
```

실행하면 자동으로 생성/갱신되는 파일:
- `records/{YYYY}/{MM}/verification_record_{시작일}_{종료일}.csv` — 날짜별 O/X 인증 기록
- `records/{YYYY}/{MM}/verification_record_{시작일}_{종료일}.json` — 메타데이터
- `records/index.json` — 드롭다운 목록 자동 갱신

#### 1-2. 오류 직접 수정

파싱 오류가 있으면 아래 파일을 직접 편집합니다.

```
records/{YYYY}/{MM}/verification_record_{시작일}_{종료일}.csv
```

열면 아래와 같은 형식입니다. `O` / `X` / `?` 값을 직접 수정합니다.

```
이름,YYYY-MM-DD,YYYY-MM-DD,...,인증률,벌금
홍길동,O,X,O,...,66%,2000원
```

#### 1-3. 벌금 납입 업데이트

납입한 멤버가 생기면 `records/payments.json`에 이름을 추가합니다.

```json
{
  "2026/04/verification_record_20260406_20260426.csv": ["홍길동", "김철수"]
}
```

- 키: `{YYYY}/{MM}/verification_record_{시작일}_{종료일}.csv` (records/ 이하 상대경로)
- 값: 납입 완료한 멤버 이름 배열 (추가할 때마다 배열에 이름 append)

---

### 2단계 — 다음 분기 시작

#### 2-1. 멤버 목록 파일 생성

```
members/{YYYY}/{MM}/members_{월}{분기}.csv
```

첫 줄은 헤더 `이름`, 이후 한 줄에 멤버 한 명씩 작성합니다.

```
이름
홍길동
김철수
...
```

폴더가 없으면 먼저 생성합니다 (`members/2026/04/` 등).

#### 2-2. 다음 분기 기록 초기화 (드롭다운 생성 + 물음표 상태)

스크립트를 새 날짜 범위로 실행하면 인증 전 상태(`?%`, `?원`)로 기록 파일이 생성되고, 드롭다운 목록(`records/index.json`)에도 자동으로 추가됩니다.

```bash
python3 check_verification.py \
  --members members/{YYYY}/{MM}/members_{월}{분기}.csv \
  --start YYYY-MM-DD \
  --end YYYY-MM-DD \
  --output records/{YYYY}/{MM}/verification_record_{시작일}_{종료일}.csv \
  --label "{월} {분기}"
```

> 채팅 파일 인자를 생략하면 모든 멤버가 `?%` / `?원` 상태로 초기화됩니다.  
> 이후 채팅이 쌓이면 같은 명령에 채팅 파일을 추가해 재실행하면 됩니다.

### 일반 공지

```
몇 가지 공지드립니다.

1. 소모임 모임통장이 있습니다. 최대한 투명하게 운영하고자 현황을 공개하고 있으며, 가입을 희망하시면 이 글에 체크 부탁드립니다.

2. 감상문 작성 시 첫 문장에 반드시 '읽'이라는 글자가 들어가야 벌금 정산 시 오류가 없습니다. 제가 직접 검토하는 게 아닌 코드로 처리하고 있으니 꼭 지켜주세요.

3. 인증 기준은 매우 널널합니다. 읽기만 했다면 간단한 사진과 감상문을 올려주시면 웬만해서는 모두 인정합니다.
   (시간 기준: 새벽 5시까지, 전날 기록으로 처리 / 예: 23일 4am 인증 → 22일로 인정)

4. 소모임 취지는 '생각하기 위해 읽는 책'입니다. 책 선택은 자유이나, 만화책·전공책 등은 인정이 어렵습니다.

궁금한 점 있으시면 편하게 말씀해주세요!
```
